Never log plain strings.</p><p>// ✅ CORRECT — Structured JSON log</p><p>logger.info({</p><p>  event: "user.login",</p><p>  userId: "usr_abc123",</p><p>  ip: "203.0.113.42",</p><p>  duration_ms: 84,</p><p>  timestamp: "2026-03-03T10:22:00Z",</p><p>  traceId: "trace-xxxx-yyyy"</p><p>});</p><p>// ❌ WRONG — Unstructured string log</p><p>console.log("User abc123 logged in from 203.0.113.42 in 84ms");</p><h2><strong>3.3 What to Always Log</strong></h2><ul><li>Authentication events — Login, logout, failed attempts, token refresh</li><li>Authorization failures — 401/403 with userId, resource, and action</li><li>All state-changing operations — Create, update, delete with before/after</li><li>External API calls — URL, method, status, duration</li><li>Database queries in slow log — Any query &gt; 100ms</li><li>Background job execution — Start, finish, errors</li><li>Deployment events — Version, environment, deployer</li></ul><h2><strong>3.4 What to NEVER Log</strong></h2><ul><li>Passwords, tokens, API keys, secrets</li><li>Full credit card numbers or bank accounts (PCI DSS compliance)</li><li>Social security numbers, passport numbers (PII)</li><li>Full request bodies with sensitive fields (mask them)</li><li>Stack traces in user-facing responses (log internally, return safe errors)</li></ul><h2><strong>3.5 Log Retention Policy</strong></h2><table><thead><tr><th><p><strong>Environment</strong></p></th><th><p><strong>Log Level</strong></p></th><th><p><strong>Retention Period</strong></p></th><th><p><strong>Storage</strong></p></th></tr></thead><tbody><tr><td><p>Production</p></td><td><p>INFO and above</p></td><td><p>90 days hot, 1 year cold</p></td><td><p>Elasticsearch + S3 Glacier</p></td></tr><tr><td><p>Staging</p></td><td><p>DEBUG and above</p></td><td><p>30 days</p></td><td><p>Elasticsearch</p></td></tr><tr><td><p>Development</p></td><td><p>All levels</p></td><td><p>7 days</p></td><td><p>Local / single node</p></td></tr><tr><td><p>Security/Audit logs</p></td><td><p>All</p></td><td><p>7 years (compliance)</p></td><td><p>Immutable S3 / Compliance vault</p></td></tr></tbody></table><h1><strong>4.
